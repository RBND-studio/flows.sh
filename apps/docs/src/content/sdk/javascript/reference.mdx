## @flows/js

Flows JS SDK is a lightweight library that allows you to integrate Flows into any application that runs JavaScript.

Install the package [from npm](https://www.npmjs.com/package/@flows/js):

```package-install
npm i @flows/js
```

### Methods

#### init()

Initializes the `@flows/js` SDK and identifies the user. This method must be called at least once in your application.

```js title="main.js"
import { init } from "@flows/js";

init({
  organizationId: "your-organization-id", // Find this in Settings > General
  environment: "production", // Default environment
  userId: "your-user-id", // Identify the user
});
```

Parameters:

| Name              | Type                              | Required | Notes                                                                                                                                                                                                                                          |
| ----------------- | --------------------------------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `organizationId`  | `string`                          | Yes      | Your organization ID, available in **Settings > General**.                                                                                                                                                                                     |
| `environment`     | `string`                          | Yes      | The environment key, available in **Settings > Environments**.                                                                                                                                                                                 |
| `userId`          | `string`                          | Yes      | Unique ID for identifying the user.                                                                                                                                                                                                            |
| `userProperties`  | `object`                          | No       | Object with custom [user properties](/users/properties) as key-value pairs. Accepts `string`, `number`, `boolean`, or `date`.                                                                                                                  |
| `language`        | [LanguageOption](#languageoption) | No       | The user's language used for [localization](/localization). When empty the user will receive the default language group.                                                                                                                       |
| `apiUrl`          | `string`                          | No       | Custom API URL for proxying requests through your domain.                                                                                                                                                                                      |
| `debug`           | `boolean`                         | No       | Enable [debug mode](/sdk/debug) to inspect the SDK's behavior.                                                                                                                                                                                 |
| `onDebugShortcut` | `function`                        | No       | Custom keyboard shortcut handler for opening the debug panel. By default, the debug panel opens with `Cmd + Option + Shift + F` on Mac or `Ctrl + Alt + Shift + F` on Windows/Linux. If you set this prop, the default shortcut will not work. |

#### addFloatingBlocksChangeListener()

Registers a listener for updates to floating workflow and tour blocks. Slottable blocks are not included.

Parameters:

| Name       | Type       | Required | Notes                                                                |
| ---------- | ---------- | -------- | -------------------------------------------------------------------- |
| `listener` | `function` | Yes      | Callback invoked with the updated [ActiveBlock](#activeblock) array. |

Returns:

A function to unsubscribe the listener and avoid memory leaks.

#### startWorkflow()

Starts a workflow for the current user in the current environment using a the block key of a manual start block. The workflow will start only if:

- Workflow is published in the current environment
- Workflow has a manual start block with a matching block key
- The current user can access the workflow based on the frequency setting
- The current user matches the start block's user property conditions

```js
import { startWorkflow } from "@flows/js";

startWorkflow("your-manual-start-block-key");
```

Parameters:

| Name       | Type     | Required | Notes                                                                                                        |
| ---------- | -------- | -------- | ------------------------------------------------------------------------------------------------------------ |
| `blockKey` | `string` | Yes      | The block key of the manual start block you want to trigger. [Learn more](/blocks/start-blocks/manual-start) |

Returns: `Promise<void>`

---

#### resetWorkflowProgress()

The `resetWorkflowProgress()` function resets the progress of a specific workflow for the current user in the current environment.

```js
import { resetWorkflowProgress } from "@flows/js";

document.querySelector("#my-button").addEventListener("click", () => {
  resetWorkflowProgress("workflow-id");
});
```

Parameters:

| Name         | Type     | Required | Notes                                                                                     |
| ------------ | -------- | -------- | ----------------------------------------------------------------------------------------- |
| `workflowId` | `string` | Yes      | The ID of the workflow to reset. You can find it in the Flows app in the workflow detail. |

---

#### resetAllWorkflowsProgress()

The `resetAllWorkflowsProgress()` function resets the progress of all workflows for the current user in the current environment.

```js
import { resetAllWorkflowsProgress } from "@flows/js";

document.querySelector("#my-button").addEventListener("click", () => {
  resetAllWorkflowsProgress();
});
```

Parameters: None

---

#### fetchWorkflows()

Returns all available workflows for the current user. Before calling this method, the `init()` method must be called first. Only enabled workflows are returned.

```js
import { fetchWorkflows } from "@flows/js";

const { workflows } = await fetchWorkflows();
```

Parameters: None

Returns: `Promise<`[WorkflowsResponse](#workflowsresponse)`>`

### Types

#### ActiveBlock

Represents a block visible to the user.

| Name        | Type                                                                                                                            | Notes                                                                                                                      |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `id`        | `string`                                                                                                                        | Internal identifier for the block. Changes with every workflow version, for persistent identifier use `props.__flows.key`. |
| `type`      | `"component" \| "tour-component" \| "survey"`                                                                                   | The block type.                                                                                                            |
| `component` | `string`                                                                                                                        | The UI component that should be used to render the block.                                                                  |
| `props`     | [ComponentProps](#componentprops) \| [TourComponentProps](#tourcomponentprops) \| [SurveyComponentProps](#surveycomponentprops) | Props passed to the component, including data and exit node callbacks.                                                     |

---

#### ComponentProps

Represents the props passed to a component. In addition to your custom props, it includes the `__flows` object with Flows properties.

| Name      | Type                                | Required | Notes                                             |
| --------- | ----------------------------------- | -------- | ------------------------------------------------- |
| `__flows` | [FlowsProperties](#flowsproperties) | Yes      | Object containing Flows properties for the block. |

---

#### StateMemory

Represents the [state memory](/blocks/block-properties/state-memory) property of a block.

| Name       | Type       | Required | Notes                                                             |
| ---------- | ---------- | -------- | ----------------------------------------------------------------- |
| `value`    | `boolean`  | Yes      | The value of the state memory.                                    |
| `setValue` | `function` | Yes      | Callback to set the value of the state memory.                    |
| `triggers` | `array`    | Yes      | Object with triggers to update the state memory. See table below. |

`triggers` type:

| Name       | Type     | Required | Notes                                                      |
| ---------- | -------- | -------- | ---------------------------------------------------------- |
| `type`     | `string` | Yes      | The trigger type. Can be `manual` or `transition`.         |
| `blockId`  | `string` | Yes      | The block ID of the block that triggers the state memory.  |
| `blockKey` | `string` | No       | The block key of the block that triggers the state memory. |

---

#### TourComponentProps

Represents the props passed to a tour component. In addition to your custom props, it includes tour component exit nodes and the `__flows` object with Flows properties.

| Name       | Type                                | Required | Notes                                             |
| ---------- | ----------------------------------- | -------- | ------------------------------------------------- |
| `continue` | `function`                          | Yes      | Callback to go to the next step.                  |
| `previous` | `function`                          | No       | Callback to go back to the previous step.         |
| `cancel`   | `function`                          | Yes      | Callback to cancel the tour.                      |
| `__flows`  | [FlowsProperties](#flowsproperties) | Yes      | Object containing Flows properties for the block. |

---

#### SurveyComponentProps

Represents the props passed to a survey component. In addition to your custom props, it includes the `survey` object, survey exit node callbacks, and the `__flows` object with Flows properties.

| Name                    | Type                                | Required | Notes                                                                                               |
| ----------------------- | ----------------------------------- | -------- | --------------------------------------------------------------------------------------------------- |
| `survey`                | [Survey](#survey)                   | Yes      | Survey data and helper methods for reading answers, navigating questions, and submitting responses. |
| Custom block properties | Depends on the property type        | Yes      | The properties of the selected block in the workflow. The properties are defined on in the block.   |
| `complete`              | `function`                          | Yes      | Callback to exit the survey block with the `complete` exit node.                                    |
| `cancel`                | `function`                          | Yes      | Callback to exit the survey block with the `cancel` exit node.                                      |
| `__flows`               | [FlowsProperties](#flowsproperties) | Yes      | Object containing Flows properties for the block.                                                   |

---

#### Survey

Represents the survey object provided to survey components.

| Name                      | Type                                  | Required | Notes                                                                                                               |
| ------------------------- | ------------------------------------- | -------- | ------------------------------------------------------------------------------------------------------------------- |
| `questions`               | [SurveyQuestion](#surveyquestion)`[]` | Yes      | Ordered list of all survey questions.                                                                               |
| `getCurrentQuestionIndex` | `function`                            | Yes      | Returns the zero-based index of the currently displayed question. Method for optional built-in question navigation. |
| `nextQuestion`            | `function`                            | Yes      | Advances to the next question and returns the new index. Method for optional built-in question navigation.          |
| `previousQuestion`        | `function`                            | Yes      | Moves to the previous question and returns the new index. Method for optional built-in question navigation.         |
| `submit`                  | `function`                            | Yes      | Submits the survey response. Use `complete()` separately to close the survey block.                                 |

#### Workflow

Represents a workflow and its state for the current user.

| Name              | Type                                    | Required | Notes                                                  |
| ----------------- | --------------------------------------- | -------- | ------------------------------------------------------ |
| `id`              | `string`                                | Yes      | The ID of the workflow.                                |
| `workflow_status` | [WorkflowStatus](#workflowstatus)       | Yes      | The publish status of the workflow.                    |
| `frequency`       | [WorkflowFrequency](#workflowfrequency) | Yes      | How often the workflow can be entered.                 |
| `user_state`      | [WorkflowUserState](#workflowuserstate) | Yes      | The current user's state in the workflow.              |
| `entered_at`      | `string`                                | No       | ISO date string of when the user entered the workflow. |
| `exited_at`       | `string`                                | No       | ISO date string of when the user exited the workflow.  |

---

#### WorkflowStatus

The publish status of a workflow. Returned in the [Workflow](#workflow) object.

- `"enabled"` - The workflow is published and active.
- `"launchpad-enabled"` - The workflow is published, active, and inside an active launchpad group.

---

#### WorkflowFrequency

How often a workflow can be entered by a user. Returned in the [Workflow](#workflow) object.

- `"once"` - The workflow can only be entered once.
- `"every-time"` - The workflow can be entered every time.

---

#### WorkflowUserState

The current user's state in a workflow. Returned in the [Workflow](#workflow) object.

- `"not-started"` - The user has not entered the workflow.
- `"in-progress"` - The user is currently in the workflow.
- `"completed"` - The user has completed the workflow.
- `"stopped"` - The user has been stopped in the workflow (e.g., by a workflow migration).

---

#### LanguageOption

Language type used for setting the user's language in the `init()` function to enable [localization](/localization).

Options:

- `disabled` (default) - The user will be served content in the default language group of your organization.
- `automatic` - Automatically detect the user's language and use the matching language group. The language is determined by the `Navigator.language` property in the browser.
- Manual - A specific language string, e.g. `en-US`, `fr-FR`, etc. This will use the matching language group for the specified language. See [https://www.localeplanet.com/icu/](https://www.localeplanet.com/icu/) for a full list of supported languages.

## @flows/js-components

To simplify implementation, you can use the `@flows/js-components` package, which provides pre-built UI components and utilities for block rendering.

Install the package [from npm](https://www.npmjs.com/package/@flows/js-components):

```package-install
npm i @flows/js-components
```

### @flows/js-components methods

#### setupJsComponents()

Defines custom elements for `flows-floating-blocks`, `flows-slot` and all the passed UI components.

Parameters:

| Name               | Type     | Required | Notes                                |
| ------------------ | -------- | -------- | ------------------------------------ |
| `components`       | `object` | Yes      | Components to use in your workflows. |
| `tourComponents`   | `object` | Yes      | Components to use in tour blocks.    |
| `surveyComponents` | `object` | Yes      | Components to use in survey blocks.  |

```js
import { setupJsComponents } from "@flows/js-components";

import * as components from "@flows/js-components/components";
import * as tourComponents from "@flows/js-components/tour-components";
import * as surveyComponents from "@flows/js-components/survey-components";

setupJsComponents({
  components: { ...components },
  tourComponents: { ...tourComponents },
  surveyComponents: { ...surveyComponents },
});
```

---

#### flows-floating-blocks

A custom HTML element for rendering floating workflow and tour blocks. For this custom element to function correctly, you need to call `setupJsComponents()`.

```html title="index.html"
<body>
  <!-- Your app content -->
  <!-- [!code highlight] -->
  <flows-floating-blocks></flows-floating-blocks>
</body>
```

---

#### flows-slot

A custom HTML element for rendering slot blocks. Use the `data-slot-id` attribute to specify the slot. For this custom element to function correctly, you need to call `setupJsComponents()`.

```html title="page.html"
<body>
  <!-- ... -->
  <flows-slot data-slot-id="my-slot">
    <!-- Only one child element with `data-placeholder` attribute is supported -->
    <div data-placeholder>
      <p>Optionally pass placeholder content here</p>
    </div>
  </flows-slot>
  <!-- ... -->
</body>
```
