Some Flows components (like [Card](/blocks/basics-v2/card)) are designed to be embedded directly in your app's UI rather than floating on top of it. These are called **slottable components**. Instead of appearing as overlays, they render inside named [slots](/blocks/block-properties/slot) you place in your layout, giving you precise control over where each component shows up.

This guide walks you through adding a slot to your app and using the built-in Card component to display content inside it.

<Steps>

<Step>

## Add a slot to your app

A slot is a placeholder in your app's markup that tells Flows where to render a component. Give it a unique ID, you'll use this same ID later in the Flows editor.

```tsx tab="React" lineNumbers
import { FlowsSlot } from "@flows/react";

const Sidebar = () => {
  return (
    <div>
      <h1>My app</h1>
      <FlowsSlot id="my-slot" />
    </div>
  );
};
```

```html tab="JavaScript" lineNumbers
<body>
  <!-- Your app content -->
  <flows-slot data-slot-id="my-slot"></flows-slot>
</body>
```

You can add as many slots as you like, each with its own ID. Place them anywhere in your layout - sidebars, dashboards, empty states, or between sections.

<Callout type="warning">
  If you're using JavaScript, `flows-slot` is a custom element that only works after you call
  `setupJsComponents()` during initialization. Without it, the slot will be present in the DOM but
  nothing will render. See the [JavaScript SDK reference](/sdk/javascript/reference#flows-slot) for
  setup instructions.
</Callout>

</Step>

<Step>

## Create a workflow

[Workflows](/workflows/overview) are where you define user experiences. Each workflow contains the logic and content for your embedded component.

To create a new workflow:

1. Go to the Flows dashboard and click **Workflows** in the sidebar.
2. Create a new workflow.

![Create workflows](/guides/embed-components/create-workflow.png)

You'll start with an empty canvas. Add a **Start** block to define when users enter the workflow. For now, leave the Start block without any user property filters so that any user can enter the workflow.

![Add start block](/guides/embed-components/add-start.png)

</Step>

<Step>

## Add a Card block

Click the blue **+** under the Start block and select **Card** from the list of available blocks. The Card block will automatically connect to Start.

![Add card block](/guides/embed-components/add-card.png)

</Step>

<Step>

## Configure the Card

With the Card block selected, you can customize its content in the properties panel:

- **Title** - the heading shown on the card
- **Body** - supporting text, accepts HTML
- **Primary button** - label and action when clicked (e.g. continue, link to a page)
- **Secondary button** - optional second action
- **Dismissible** - shows a close button if enabled

![Card properties](/guides/embed-components/card-properties.png)

See the [Card component docs](/blocks/basics-v2/card) for a full list of properties.

</Step>

<Step>

## Set the slot

Still in the Card block's properties panel, find the **Slot** section. Set the **Slot ID** to match the ID you used in your code - in this example, `my-slot`.

![Slot configuration](/guides/embed-components/slot.png)

<Callout>
  Without a Slot ID set, the Card will not be rendered. The slot must also exist in your app's code
  before it can be targeted.
</Callout>

If you're placing multiple blocks in the same slot, use the **Slot index** to control the order. Blocks with a lower index render first.

Learn more about the [Slot property](/blocks/block-properties/slot).

</Step>

<Step>

## Test your workflow

Before publishing to real users, verify that the card appears in the correct location.

1. Click **Publish** in the top-right corner.
2. In the publish dialog, select a **staging environment** to limit exposure during testing.
3. Visit your app and check that the card appears where you placed the slot.

To reset the workflow and run through it again:

1. Open the **Users** page in Flows.
2. Find your user profile and open the **Workflows** tab.
3. Locate the workflow and click **Reset progress**.
4. Refresh your app.

Learn more about [testing workflows](/workflows/testing).

![Card in app](/guides/embed-components/demo.png)

</Step>

<Step>

## Publish to users

Once you're happy with the result, publish to production.

1. Optionally configure user property filters in the **Start** block to target a specific segment. See the guide on [targeting only new users](/guides/how-to-target-only-new-users) for an example.
2. Click **Publish** and select the **production** environment.

</Step>

</Steps>

## Using custom components in slots

The Card component is a quick way to get started, but you can also build your own slottable components with custom UI and logic. See [Create custom components](/guides/create-custom-components) to learn how.
